## 1. 引言

### 1.1 目的

本需求规格说明书（Software Requirements Specification，SRS）旨在详细描述"牙周临床教学平台"的功能需求、非功能需求及相关约束条件，为系统设计、开发、测试和验收提供基准依据。

本需求规格说明书遵循ISO/IEC/IEEE 29148:2018《系统与软件工程——生命周期过程——需求工程》标准编制，确保需求描述的完整性、一致性、可验证性和可追溯性。

### 1.2 范围

#### 1.2.1 系统名称

牙周临床教学平台（Periodontal Clinical Teaching Platform，简称PCTP）

#### 1.2.2 系统概述

牙周临床教学平台是一个面向口腔医学专业师生的综合性教学管理系统，由**教师端系统**和**学生端系统**两个独立子系统组成。平台采用独立账号体系，支持教师、学生、管理员和游客四种角色，实现牙周临床教学的数字化、智能化和个性化。

## 2. 总体描述

### 2.1 用户角色

| 角色 | 描述 | 可访问系统 |
|------|------|------------|
| 教师 | 口腔医学专业教师 | 教师端系统 |
| 学生 | 口腔医学专业学生 | 学生端系统 |
| 管理员 | 系统管理人员 | 教师端系统、学生端系统、管理后台 |
| 游客 | 未登录访客 | 学生端系统（部分功能） |

### 2.2 约束条件

#### 2.2.1 技术约束

| 约束项 | 约束内容 |
|--------|----------|
| 架构模式 | B/S架构，Web应用 |
| 客户端 | 支持主流浏览器（Chrome、Firefox、Edge、Safari） |
| 移动端 | **[AI补充]** 当前版本不支持移动端适配，后续迭代考虑 |
| 认证方式 | 独立账号体系，不支持单点登录(SSO) |
| 外部系统 | 外链系统需用户自行完成二次登录认证 |

#### 2.2.2 业务约束

| 约束项 | 约束内容 |
|--------|----------|
| 数据范围 | 历史数据查询范围为本学期 |
| 内容审核 | 虚拟学术大厅发布内容无需审核 |
| 文件上传 | 支持文件、图片、文字格式，不限制文件大小 |
| 功能扩展 | 禁止添加客户需求未提及的功能点 |

### 2.3 假设与依赖

| 编号 | 假设/依赖内容 |
|------|---------------|
| A-001 | 用户具备基本的计算机操作能力和网络访问条件 |

| D-001 | 外部链接系统的网络可达性 |
| D-002 | AI辅助诊断系统文件夹内容提供（后续迭代） |
